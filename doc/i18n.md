# Internationalization (i18n)

The project now uses a **dynamic i18n architecture**:

- default locale (`vi`) lives at root paths (`/`, `/about`, `/entries/...`)
- all locale pages are generated from a single dynamic route tree: `src/pages/[...lang]/...`
- routing helpers and content loading are locale-generic (no hardcoded `en`)

Localized paths in components use [`src/i18n/paths.ts`](../src/i18n/paths.ts):

- `localePath(lang, path)`
- `localeFromPathname(pathname)`
- `alternateLocalePath(pathname, targetLocale)`
- `localeStaticPaths()`

## Locales & URL Strategy

| Setting | Value |
|--------|--------|
| Default locale | `vi` |
| Supported locales | `locales` array in `src/i18n/config.ts` |
| Root pages | unprefixed (`/`, `/about`, `/entries/...`) for default locale |
| Other locales | prefixed (`/{lang}/...`) generated by `src/pages/[...lang]/...` |

Legacy redirects (`/vi`, `/vi/*`) still point to unprefixed default paths.

## Dynamic Route Layout

- Unified locale routes:
  - `src/pages/[...lang]/index.astro`
  - `src/pages/[...lang]/about.astro`
  - `src/pages/[...lang]/entries/**`

`[...lang]` routes use `getStaticPaths()` for all locales, with `lang: undefined` for default locale and `lang: "<locale>"` for prefixed locales.

## Content Collections

File: `src/content.config.ts`

- Collections are generated dynamically from `locales` as `entries{Locale}`
- Example keys: `entriesVi`, `entriesEn`, `entriesJa` (when locale exists)
- Markdown source stays `src/content/{locale}/entries/**/*.md`
- Shared schema: `entrySchema`

## Fallback behavior

File: `src/i18n/content.ts`

- `getLocalizedEntries(locale)` loads locale collection and falls back to default-locale entries by `id` when missing
- `getLocalizedEntry(locale, id)` falls back to default locale for missing localized file
- `getAllEntryIds()` reads canonical IDs from default locale

## UI strings and category labels

- UI copy: `src/i18n/config.ts` (`ui[locale][key]`, `t(locale, key)`)
- Category labels: `src/data/category-labels.ts` (`Record<slug, Record<Locale, string>>`)

## Language switch and SEO alternates

- `src/components/Header.astro` loops over `locales` for switch links
- `src/layouts/BaseLayout.astro` and `src/layouts/EntryLayout.astro` emit hreflang alternates for all locales in `locales`

## Dynamic-First Policy (for future changes)

All future i18n features must follow these rules:

1. Do not add mirrored page trees like `src/pages/en/*`, `src/pages/ja/*`
2. Add locale support through `locales` and existing dynamic routes/helpers
3. Keep path + fallback logic generic for every locale in `locales`
4. If a change introduces locale-specific duplication, refactor it before merging
