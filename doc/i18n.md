# Internationalization (i18n)

The site uses **Astro i18n with unprefixed default locale**: Vietnamese (`vi`, default) has URLs at the site root (`/`, `/about`, `/entries/...`). English (`en`) uses the **`/en/`** prefix (`/en/`, `/en/about`, `/en/entries/...`). All user-facing UI strings are keyed in `src/i18n/config.ts`; markdown bodies come from locale-specific content collections with **English fallback to Vietnamese** when an EN file is missing.

Localized paths in components use **`localePath(lang, path)`** from [`src/i18n/paths.ts`](../src/i18n/paths.ts). The language switch uses **`alternateLocalePath(pathname, targetLocale)`**.

## Locales & URLs

| Setting | Value |
|--------|--------|
| Default locale | `vi` |
| Locales | `vi`, `en` |
| Root `/` | Vietnamese home (`src/pages/index.astro` → `HomePage`) — **not** a redirect |

**URL patterns** (trailing slash ignored by Astro config):

| Pattern | Page |
|--------|------|
| `/`, `/about`, `/entries`, `/entries/category/[slug]`, `/entries/[id]` | Vietnamese |
| `/en/`, `/en/about`, `/en/entries`, `/en/entries/category/[slug]`, `/en/entries/[id]` | English |

**Legacy:** `/vi` and `/vi/*` → 301 to `/` and the corresponding unprefixed path (`astro.config.mjs` + `public/_redirects` for Netlify-style hosts).

Routing is implemented with **root pages for `vi`** and **`src/pages/en/`** for English, plus Astro `i18n` in `astro.config.mjs` with `prefixDefaultLocale: false`.

### Static locale pages (no markdown collection)

`src/pages/about.astro` (VI) and `src/pages/en/about.astro` (EN) use `t(lang, '...')`. Example key prefix: **`about.*`**. Add every new key under both `vi` and `en` in `src/i18n/config.ts`.

## Content collections

File: `src/content.config.ts`

- **`entriesVi`** — `glob` on `./src/content/vi/entries/**/*.md`
- **`entriesEn`** — `glob` on `./src/content/en/entries/**/*.md`
- Shared Zod schema: `entrySchema` (same fields for both)

Entry `id` is the filename without `.md` (e.g. `lac-long-quan`). IDs align across locales so the lang switch maps `/entries/...` ↔ `/en/entries/...` for the same story.

## Fallback behavior

File: `src/i18n/content.ts`

- **`getLocalizedEntries(locale)`** — For `en`, merges published EN entries with published VI entries whose `id` is missing from EN (so the catalog lists every story).
- **`getLocalizedEntry(locale, id)`** — For `en`, if no EN entry exists for `id`, returns the VI entry.
- **`getAllEntryIds()`** — Published IDs from the VI collection (canonical list for path generation).

Detail and list pages use these helpers so `/en/entries/...` never 404s for a story that exists only in Vietnamese.

## UI strings

File: `src/i18n/config.ts`

- **`ui[locale][key]`** — Nav, hero, footer, entry labels, About page (`about.*`), etc.
- **`t(locale, key)`** — Resolves a string; falls back to `vi`, then to the key.

Layouts and components receive a **`lang`** prop (`Locale`) and call `t(lang, ...)` where needed.

## Category labels

File: `src/data/category-labels.ts`

- **`CATEGORY_LABELS`** — `Record<slug, Record<Locale, string>>`
- **`getCategoryLabel(slug, locale)`** — Returns label for the slug in the active locale, with fallback to `vi` then raw slug.

## Language switch

File: `src/components/Header.astro`

- Uses `alternateLocalePath(Astro.url.pathname, 'vi' | 'en')` so `/about` ↔ `/en/about`, `/` ↔ `/en/`, etc.
- VI and EN are plain `<a>` links (no client JS).

## HTML `lang` attribute

`src/layouts/BaseLayout.astro` and `src/layouts/EntryLayout.astro` set `<html lang={lang}>` from the active locale.

## Adding a new locale (outline)

1. Add the locale to `locales` in `src/i18n/config.ts` and to `astro.config.mjs` → `i18n.locales`.
2. Add a collection in `src/content.config.ts` and a folder under `src/content/{locale}/entries/`.
3. Extend `getLocalizedEntries` / `getLocalizedEntry` if you need fallback rules.
4. Add UI keys for `ui[newLocale]` (copy from `vi` as template).
5. Add a `src/pages/{locale}/` tree mirroring `en/` and extend `localePath` in `src/i18n/paths.ts`.
